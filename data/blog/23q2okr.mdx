## 起因

最近公司開始實施 OKR，為了學習自己去看了 Radical Focus 這本 OKR 的書。看完其實有蠻多啟發，包含怎麼用 OKR 讓團隊更專注，怎麼樣設計這些 OKR 才真正有效果，於是想要自己也來嘗試用在個人的計畫上。
我一直以來都是一個很不專注的人，字面上還有規劃上，很常分心，或是一直把精力放在一些不重要的地方。所以希望這次能用 OKR 來幫助我達成我自己 Q2 的一些目標。

這篇紀錄就是其中一個 Objective，比起一個完整的文章可能更像是一個日記形式的輸出紀錄

我幫自己訂的第一個 OKR 如下

O: Get the output machine oiled and running

- KR1: daily output commit
  - (Can be anything, but needs to be out)
- KR2: Finish nights & weekends and share experience
- KR3: Get 5000 exposure & 100 interactions (fuel for the machine)

主要的目標就是打造一個輸出的習慣還有系統，輸入的再多還是要動手才能真正吸收。而輸出就是一個很好的方法驅動整個輸入->動手->輸出的系統，有更明確的結果，也有更好的回饋機制。

第一個 KR 就是這個紀錄，我要每天都在這裡有輸出，可能是指向另一個輸出的指引，也可能就是單純的紀錄，無論如何，要上傳上來。話是這麼說，老實說從我下決定到現在我也拖了好幾天，可能要回頭補一下。

第二個 KR 則是我最近參與的一個 program buildspace nights & weekends，簡單來說就是六週，大家來做自己想做的 Project，他會有些 check point 來督促你推進進度。完整參與就需要一定的輸出（大部分是動手的部分），剛好很適合拿來當 KR。

第三個則是一個更客觀的衡量，但同時也追求一個正向循環，越多處及越多人看到，應該會有更多動力。

## 每日紀錄

### 04/23 補紀錄

不知道大家用 ChatGPT 的時候，有沒有這樣的困擾：「用聊天的形式學東西，很常需要來回翻動，有時候見樹不見林」？
我自己很喜歡用知識圖譜、視覺化的方式學東西，受筆記軟體 Heptabase 啟發，想說如果能讓 AI 幫我生成這種卡片知識圖，應該可以跳脫線性的方式，更自由地探索。
所以我做了一個小小網站 AI Knowlege Map，把 OpenAI 的 API 串進了可視化的白板上，可以從一個主題開始，無限發散、收斂，最後產出自已的知識圖(knowledge map)
在我 API 預算爆掉之前，歡迎大家免費無限試用！
網站是個原型，所以可能有不少 bug，等回應也要可能要 20-30 秒，請大家多多擔待 (Demo 影片是剪接過的 lol
如果你用了之後遇到什麼問題或有什麼建議，歡迎跟我說，非常感謝
自己一個人做 side project 常常會偷懶沒動力，最近 apply 了一個叫 buildspace 的 program，六週要把 idea 做出來。每週也有作業，這週要有 10 個 user，所以大家盡量用 😂 。看到其他人一直有產出，自己也有動力了一些。這個網站就是我這六週的 idea，看看能做出些什麼好玩的東西。

### 04/27

這兩天小小的分心，莫名其妙地做了一個棒球發球機。看完大谷翔平的比賽後，就開始突然很想丟棒球，不過剛好女友不在，而且我也不指望他跟我傳接球，畢竟幾週前我還在跟他解釋好球壞球。想一想只好自立自強，搞一個傳接球機。

週末開始計劃，週間準備了一下材料，3D 列印了一個劣質的轉接頭，終於做出了一個基本的樣子。主要結構是木頭，用的動力是一個電動滑板用的馬達，其實很有力，速度也可以到很快。配合水管，讓輪子把送出去，就是基本概念。

要傳接球的話又需要另一個接球網，讓球自動滾回去餵下一顆。這兩天來回了一下，可以丟出一定速度的球，但是有幾個小問題

- 輪子連接馬達不夠穩固，太快的速度（超過 50%)就會開始震動。
- 輪子可能不夠大，現在用的輪子有點太小，能幫球加速的距離確實讓能量沒辦法轉換更好
- 接球的部分需要不少的結構支撐，才能把球接好，不然也不敢太大力丟

詳細內容在我的[虛擬車庫](https://www.notion.so/joeywangtw/Baseball-launcher-8c4ae7eeb95e47c2bfb2063ece59cd66?pvs=4)裡

### 04/28

前一陣子聽了一些[Andrew Huberman](https://www.youtube.com/@hubermanlab)的 podcast，其中一個主題是在說「如何保持動力」，其中有一點是說要合理地控制多巴胺水平，太多的起伏反而會導致過度刺激之後的下墜，讓整體多巴胺低於平均而失去動力。

這週的我就有點這個感覺，上週工作有些空檔，我就在下班後多做了不少我想做的 side project，整體來說是穩定推進那種。但是到了週末，我就切換成了廢寢忘食的模式。確實做出了一些產出（我做了我的 AI 知識圖譜的原型 4/23 的紀錄)，所以週末算是多巴胺相對高的，整體來說動力滿滿。

但是到了這週，上班稍微忙了點，壓力也多了點，下班後就有點沒力。雖然沒力，但是我還是不自覺的想要追尋一些多巴胺刺激，所以才會有傳接球機這個 Project，想要讓自己不無聊。所以一些我計畫好想做的事，在上班跟傳球機之後，幾乎是沒有任何動力啟動了。意識到這點，我在想或許這週末也不應該全速前進，反而應該要稍微克制一點，因為我發現每天持續地推進，不只產出更多，也更能持之以恆。這週末可能在做有趣的 project 之外，也要做點相對刺激低的事，例如打掃家裡、看書，看看能不能把我的動力拉回平均線。

### 04/29

今天整天算是輕鬆的一天，耍了不少費，開了新番水星的魔女，還邊吃飯邊看了 BlackPink 的紀錄片。當然 project 也不能閒著，我改進了一下我的發球機，試著換一個更大的輪子，好像球更強了。另外也花了些時間想想我的 Side project AI Knowledge Map，po 在我的個人頁面還有社團裡的前幾天，有一些用量，但是從一些反饋中就知道，有一些事情沒想清楚。
首先，光是建立知識圖譜這件事情本身就很難，只是不斷發散收斂，其實沒有辦法做的很好，會有很多水平的主題沒辦法連結，也很容易上下節點重複。
另外，要真正能夠快速掌握一個知識，真的很難有一個 one size fit all 的 prompt，所有的 prompt 都要更聰明，這也代表怎麼處理資料就會變得很複雜。而每次都有很不同的結果也代表要好好的排列節點會變得很複雜，這也是我得到很多的反饋，大家都在說卡片重疊了。
最後，最核心的使用者體驗問題其實在 API 的回應實在太慢了，每按一下都要等個 20-30 秒，誰受得了，這絕對不會是一個很好的學習體驗。

經過一番思考之後，我想應該要先暫時放下「快速學一個東西」這件事，先從一個更簡單、更通用的解法下手。其實要解決的問題不變，但是更簡化了，也就是解決 thread 不適合發散、收斂、自由連結這件事。先把一個很彈性的系統做出來，看大家都怎麼用，或許會得出更好的想法。第一步就是做出卡片版的 ChatGPT，一切都一樣，就是可以用卡片白板的形式操作。

講這麼多，今天的第一要務就是做出像是 ChatGPT 那種 Streaming 的形式，因為有東西在跑，可以慢慢看，如果要等到全部都出來真的太久了。OpenAI 的 API 有些有支援這種 Streaming，會用 server-sent event 來持續傳輸資料。
我參考了[Vercel 的一篇教學](https://vercel.com/blog/gpt-3-app-next-js-vercel-edge-functions#the-frontend)，講得算詳細，但是要注意他用的 API 是 completion 而不是 Chat，有些東西不太一樣，也讓我今天卡了蠻久的，最後終於做出想要的效果，讓他用 streaming 的方式講一個故事，算是搞懂了一個必要的核心功能。

可以考慮來寫一篇中文的學習紀錄，感覺這個是一個很重要的 ChatGPT API 必要條件，但是資源還是偏少。
